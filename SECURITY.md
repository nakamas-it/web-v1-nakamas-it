# Security Policy

## About This Project

This is a personal static website — a portfolio and networking protocol educational library. There is no backend, no database, and no user authentication. The site is built with Astro and hosted on GitHub Pages.

## Supported Versions

Only the current version deployed from the `main` branch is actively maintained.

| Version | Supported |
| ------- | --------- |
| `main` (latest) | ✅ |
| Any previous commit | ❌ |

## Reporting a Vulnerability

If you discover a security issue in this project — including but not limited to: exposed secrets or credentials in the repository, a misconfigured GitHub Actions workflow, a third-party dependency with a known CVE, or a cross-site scripting (XSS) issue in the site's HTML/JavaScript — please report it privately rather than opening a public GitHub issue.

**The preferred contact method is email:**

📧 [gcisneros@nakamas-it.com](mailto:gcisneros@nakamas-it.com)

Please include in your report:
- A clear description of the vulnerability
- Steps to reproduce or a proof-of-concept
- The potential impact in your assessment

You can expect an acknowledgement within **72 hours**. Given the nature of this project (a personal static site with no server-side logic or user data), most reports will be resolved quickly. If a reported vulnerability is confirmed, a fix will be deployed as soon as possible and you will be credited in the commit message unless you prefer to remain anonymous. If a report is not accepted, you will receive an explanation of why.

Thank you for helping keep this project secure.
